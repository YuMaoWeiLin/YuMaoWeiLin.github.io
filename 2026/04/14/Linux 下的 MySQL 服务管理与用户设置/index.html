三、学习中的几点理解

1. 用户名和主机要一起看

MySQL 中的用户并不是单独看用户名，而是看：

所以即使用户名相同，只要主机不同，也会被当成不同用户。

2. 创建用户不等于拥有权限

很多初学者容易把“创建用户”和“授予权限”当成一件事，但它们其实是两个步骤：

  1. CREATE USER
  2. GRANT 权限

3. % 很方便，但要注意安全

把主机写成 %，表示允许任意主机连接。

在测试环境中这样做比较方便，但如果数据库处于公网环境，风险会明显增大，因此应该谨慎使用。

四、总结

这次学习让我对 MySQL 的基础管理有了更清晰的认识。

对于 Linux 环境下的 MySQL，首先要掌握服务管理命令：

  • 启动服务
  • 停止服务
  • 重启服务
  • 查看状态

而在数据库内部，还要掌握用户管理操作：

  • 创建用户
  • 授权
  • 刷新权限
  • 查看权限
  • 撤销权限
  • 删除用户
  • 修改密码

这些内容本身不难，但都是后续学习数据库设计、权限控制和安全管理的基础。
