在最近的 MySQL 学习中，我把最常见、最基础的一批操作重新整理了一遍。相比单纯记命令，我更想弄清楚每条语句的作用、适用场景和核心理解。

这篇笔记主要覆盖以下内容：

  • 数据库与表
  • 增删改查
  • 条件筛选与排序
  • NULL 的判断
  • 表结构修改
  • 常用聚合函数
  • 多表查询与结果拼接

适合刚开始学习 MySQL 的同学快速复习。
建议学习顺序：先理解“库、表、列、行”，再记 SELECT / INSERT / UPDATE / DELETE，最后再补 JOIN 和 UNION。

一、数据库与表

数据库可以理解成一个大仓库，表就是仓库里的不同分类区域。学习 MySQL 时，先把“数据库”和“表”的概念分清楚，后面的命令会更容易理解。

1. 创建数据库

作用： 新建一个数据库。
理解： 数据库就是存放数据的大仓库。

2. 使用数据库

作用： 进入指定数据库。
提示： 如果看到 Database changed，说明切换成功。

3. 创建表

作用： 创建 students 表，并定义每一列的字段信息。

几个关键点：

  • PRIMARY KEY：主键，要求唯一
  • AUTO_INCREMENT：自动编号，通常搭配主键使用
  • NOT NULL：该列不能为空
  • VARCHAR(50)：可变长度字符串，最大长度为 50

4. 查看表结构

作用： 查看表中各列的结构信息。
重点关注：

  • Field：列名
  • Type：数据类型
  • Key：是否为主键

5. 查看当前数据库中的表

作用： 查看数据库里已经存在的所有表。

二、数据操作：增删改查

增删改查是数据库中最核心的一部分，也是最常使用的内容。

1. 插入数据

作用： 插入一整行数据。
要点： 列的顺序要和后面的值一一对应。

2. 查询所有数据

作用： 查询表中的全部数据。

3. 条件查询

作用： 按条件筛选符合要求的数据。

4. 多条件查询

要点：

  • AND：并且
  • OR：或者

5. 修改数据

作用： 修改符合条件的记录。
注意： UPDATE 一般必须带 WHERE，否则可能把整张表都改掉。

6. 删除数据

作用： 删除符合条件的记录。
注意： 不加 WHERE，可能会把整张表的数据全部删掉。

7. 清空表

作用：

  • 清空整张表的数据
  • 重置自增主键

区别： TRUNCATE 更像“整表清空”，而 DELETE 更适合按条件删除。

三、查询进阶

在掌握最基本的查询后，还需要学会模糊查询、排序和限制条数。

1. 模糊查询

作用： 查询姓名以“张”开头的数据。
要点： % 表示任意长度的任意字符。

2. 排序

要点：

  • ORDER BY age：按年龄升序排序
  • DESC：降序排序

3. 限制返回条数

作用： 只返回前两条记录。
应用： 适合测试查询结果，或者只想查看前几行数据时使用。

七、多表操作

学习到这里，就不能只看单表了。实际开发中，经常需要把多张表联系起来一起查询。

1. JOIN：横向拼接数据

作用： 根据关联字段，把两张表的数据拼接起来。

本质理解：

  • students.id 和 scores.student_id 是对应关系
  • JOIN 就是在找“谁和谁是一一对应的”
  • 最终结果是把不同表里的列放到同一行中展示

八、结果拼接

除了把表横着拼，还可以把查询结果竖着拼。

1. UNION

作用： 纵向合并两个查询结果，并自动去重。

2. UNION ALL

作用： 纵向合并两个查询结果，但不去重。

区别：

  • UNION：去重
  • UNION ALL：不去重，速度通常更快

九、核心理解总结

MySQL 入门阶段，最重要的不是死记命令，而是先把每条命令背后的含义弄清楚

1. 基本概念

  • 列：竖着的一栏，也叫字段
  • 行：横着的一条，也叫记录

2. 常用命令的本质

  • INSERT：插入一整行数据
  • SELECT：查询数据
  • WHERE：加条件筛选
  • ORDER BY：排序
  • LIMIT：限制返回条数
  • JOIN：横向拼接多表数据
  • UNION：纵向拼接查询结果

3. 我的学习体会

刚开始学 MySQL 时，最容易混淆的地方是：

  • 表结构和表数据的区别
  • DELETE 和 TRUNCATE 的区别
  • NULL 不能用 = 判断
  • JOIN 是横向拼表，UNION 是纵向拼结果

把这些关键点理顺之后，再回头看前面的语句，就会清楚很多。
