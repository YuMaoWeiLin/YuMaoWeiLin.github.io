C++基础算法之深搜的剪枝技巧

文章发布时间:
2026-03-20

最后更新时间:
2026-03-20

在使用深度优先搜索时，如果把所有情况都无脑搜索一遍，往往会非常慢。
这时就需要一种优化方法：剪枝

所谓剪枝，就是：

在搜索过程中，提前判断某些分支不可能产生最优解或合法解，于是直接停止继续搜索

这样就能大大减少搜索量，提高运行效率

一、什么是剪枝

可以把整个搜索过程想成一棵树。
如果某个分支已经确定不可能有答案，那就像把这根树枝剪掉，不再继续往下搜索

因此，剪枝的目的就是：

  • 减少无效搜索
  • 提高程序速度
  • 避免超时

二、为什么 DFS 常常需要剪枝

DFS 的特点是会把所有可能情况都试一遍
如果问题规模稍大，搜索树就会非常庞大

例如：

  • 枚举排列：n!
  • 枚举子集：2^n
  • 某些组合问题：分支数非常多

如果不剪枝，程序可能要跑很久

三、常见剪枝思路

DFS 中常见的剪枝方法有以下几种。

1. 越界剪枝

如果当前状态已经不合法，就立刻返回

例如走迷宫时，如果坐标越界，就不必继续搜

2. 重复状态剪枝

如果某个状态已经访问过，就不要再重复访问

3. 最优性剪枝

如果当前代价已经不可能优于当前最优答案，就直接停止

4. 可行性剪枝

如果从当前状态出发，无论怎么走都不可能成功，就提前停止

这种剪枝往往需要结合题目性质来判断

七、剪枝不等于乱剪

剪枝虽然能加快速度，但一定要注意：

剪枝不能把正确答案剪掉

所以写剪枝条件时必须非常谨慎，确保：

  • 被剪掉的分支一定无效
  • 或一定不可能更优

如果条件写错，程序可能会直接漏解
